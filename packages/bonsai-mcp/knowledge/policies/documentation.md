## Documentation

- ALWAYS rank any newly added future features using the star system
- ALWAYS re-sort the list when adding so that the list maintains the sort order of least effort/risk (1 star) to most effort/risk (6 stars)
- ALWAYS, when a feature is marked complete, update all affected project docs in the same change set (`docs/roadmap.md`, `docs/testing.md`, `docs/troubleshooting.md` as applicable) and set status/checklists to reflect completion.
- ALWAYS include a concise changelog-style implementation note for each completed feature (what shipped, where it was implemented, and user-visible behavior), so release notes can be generated directly from documentation history.
